# Upload Guide

## Recommended GitHub Upload Set

Upload the following items to the public repository:

- `README.md`
- `assets/diagrams/`
- `assets/screenshots/`
- `docs/asset-captions.md`

## What Not to Upload Publicly

Do **not** upload the raw original penetration test report to a public repository unless you have independently confirmed that:
1. it is fully sanitized
2. it contains no credentials, passwords, personal data, exploit-ready detail, or sensitive lab artifacts
3. sharing it is permitted under the original assignment, NDA, or course guidance

## Best Public Portfolio Positioning

Use this repository as a **sanitized case study**, not as a dump of full exploit evidence. That keeps the project professional and recruiter-friendly while still demonstrating technical depth.

## Suggested Repo Description

Sanitized penetration testing case study demonstrating service enumeration, attack-path analysis, sensitive data exposure assessment, and remediation planning in an authorized lab environment.

## Suggested GitHub Topics

`cybersecurity` `penetration-testing` `security-engineering` `vulnerability-assessment` `nmap` `linux` `security-reporting`

## If You Want to Expand This Repo Later

Good additions would be:

- a one-page findings summary table
- a MITRE ATT&CK mapping graphic
- a remediation-priority matrix
- a short `lessons-learned.md` file focused on defensive takeaways
